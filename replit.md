# QuantumPitch AI

Elite cricket analytics platform that ingests smartwatch IMU data from Galaxy Watch 6 (via Sensor Logger) and ESP32 Smart Stump signals, processes biomechanical physics, and broadcasts real-time telemetry to a live dashboard.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/quantum-pitch run dev` — run the dashboard frontend
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)
- Real-time: Pusher (channel: `live-session-0338`)
- Frontend: React + Vite, Recharts, pusher-js

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI contract (source of truth)
- `lib/db/src/schema/` — Drizzle table schemas (sessions, deliveries, wickets)
- `artifacts/api-server/src/routes/` — Express route handlers
- `artifacts/api-server/src/lib/physicsEngine.ts` — IMU biomechanics engine
- `artifacts/api-server/src/lib/pusher.ts` — Pusher broadcaster
- `artifacts/quantum-pitch/src/` — React dashboard frontend

## Architecture decisions

- Immediate HTTP 200 response on `/api/telemetry` and `/api/stump` before async processing to prevent IoT device timeouts
- Physics engine filters `WatchTotalAcceleration` sensor, applies 3G peak threshold, double-integration over swing window, and ZUPT drift correction
- Pusher used for server→client push (new-ball, wicket-alert, session events); client invalidates React Query cache on each event
- All DB tables use `TEXT` primary keys (UUIDs generated by `crypto.randomUUID()`) to avoid sequence collisions
- OpenAPI-first: all routes derived from spec, codegen produces both React Query hooks and Zod validators

## Product

- **Live Dashboard** (`/`): Start/close match sessions, real-time delivery feed with pace readings, wicket alerts, velocity trend chart, and aggregate stats
- **Session History** (`/history`): Past sessions with per-session aggregate stats
- **POST /api/telemetry**: Receives 1-second IMU batch from Galaxy Watch 6 via Sensor Logger, processes biomechanics, broadcasts `new-ball` event via Pusher
- **POST /api/stump**: Receives `{status, impact_force}` from ESP32, saves to DB, broadcasts `wicket-alert` via Pusher
- **POST /api/session/start**: Creates a new active session, broadcasts `session-started`

## User preferences

- Target hardware: Galaxy Watch 6 (Sensor Logger) + ESP32 Smart Stump
- Pusher cluster: ap2

## Gotchas

- The sessions table was pre-existing with old columns — was dropped and recreated with the correct schema
- `drizzle-kit push` is interactive; use `executeSql` directly for schema changes
- OpenAPI response schema names must NOT match Orval's auto-derived `<OperationId>Response` pattern or TS2308 collision occurs
